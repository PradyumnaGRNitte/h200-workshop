# Contributing to H200 Workshop Materials

Thank you for your interest in improving the workshop materials!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest improvements
- Include your Python version, PyTorch version, and error messages
- Describe what you expected vs. what happened

### Suggesting Enhancements
- Open an issue describing the enhancement
- Explain why it would be useful for workshop participants
- Provide examples if possible

### Submitting Code
1. Fork the repository
2. Create a feature branch: `git checkout -b feature-name`
3. Make your changes
4. Test thoroughly on the H200 cluster
5. Submit a pull request

## Code Standards

- Follow PEP 8 style guidelines
- Add comments for complex sections
- Update documentation if needed
- Test on actual H200 hardware

## Workshop Materials Guidelines

- Keep examples simple and clear
- Focus on educational value
- Ensure code runs on H200 MIG slices (1g.18gb)
- Update timing estimates if you change code
